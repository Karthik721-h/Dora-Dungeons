FEATURE: FULL RESPONSIVE UI SYSTEM

Goal:
Make the entire game responsive, organized, and accessible across:
- Desktop (≥1024px)
- Tablet (768px–1023px)
- Mobile (≤767px)

DO NOT:
- Break existing game logic
- Modify API behavior
- Affect voice system

-------------------------------------

STEP 1 — DEFINE BREAKPOINT SYSTEM

Use standard breakpoints:

Mobile: ≤767px
Tablet: 768px–1023px
Desktop: ≥1024px

If using Tailwind:
- sm, md, lg breakpoints

-------------------------------------

STEP 2 — MAIN LAYOUT STRUCTURE

Create a responsive layout container:

- Header (top nav)
- Game content (center)
- Status panel (side or stacked)

Desktop:
→ 3-column layout:
   [Sidebar | Game | Status]

Tablet:
→ 2-column:
   [Game | Status]
   Sidebar collapses

Mobile:
→ Single column:
   [Header]
   [Game]
   [Status collapsible]

-------------------------------------

STEP 3 — HEADER (NAV BAR)

Fix issues:

- Prevent overflow
- Ensure dropdowns are visible (z-index fix)

Mobile:
→ Stack elements OR use compact layout
→ Replace text with icons where needed

Ensure:
- Voice dropdown visible (z-index: 9999)
- Buttons spaced properly

-------------------------------------

STEP 4 — TERMINAL / GAME LOG

Make terminal responsive:

- Use max-height with scroll
- Font-size scales down on mobile
- Add padding for readability

Mobile:
→ Reduce font slightly
→ Increase line spacing

-------------------------------------

STEP 5 — STATUS PANEL

Desktop:
→ Fixed right panel

Tablet:
→ Smaller side panel

Mobile:
→ Convert to collapsible section:

Button:
"View Status"

Tap:
→ Expand panel

-------------------------------------

STEP 6 — MODALS (CRITICAL)

Ensure ALL modals:

- Centered properly
- Max width: 90% on mobile
- Padding adjusted

Fix:
- No overflow
- Buttons stack vertically on mobile

-------------------------------------

STEP 7 — BUTTONS & TOUCH TARGETS

Minimum touch size:
→ 44px height

Spacing:
→ margin between buttons

Mobile:
→ Stack buttons vertically

-------------------------------------

STEP 8 — SHOP UI

Fix shop layout:

Desktop:
→ Grid (2–3 columns)

Tablet:
→ 2 columns

Mobile:
→ 1 column (stacked buttons)

Ensure:
- No overflow
- Prices readable
- Buttons full width on mobile

-------------------------------------

STEP 9 — INTRO VIDEO

Ensure:

- Fullscreen on all devices
- object-fit: cover

Mobile:
→ No black bars
→ Proper scaling

-------------------------------------

STEP 10 — DROPDOWNS (IMPORTANT BUG FIX)

Fix dropdown visibility:

- Use position: absolute
- Add z-index: 9999
- Ensure parent has position: relative

Mobile:
→ Make dropdown full width if needed

-------------------------------------

STEP 11 — SCROLL MANAGEMENT

Ensure:

- No horizontal scrolling
- Only vertical scroll where needed

Fix:
overflow-x: hidden

-------------------------------------

STEP 12 — TYPOGRAPHY

Responsive font sizes:

Desktop:
→ Normal

Tablet:
→ Slightly reduced

Mobile:
→ Smaller but readable

Ensure:
- High contrast (for accessibility)
- Clear hierarchy

-------------------------------------

STEP 13 — VOICE UI

Ensure:

- Voice button always visible
- No overlap with other UI
- Accessible on all screen sizes

-------------------------------------

STEP 14 — TEST SCENARIOS

Desktop:
→ Everything aligned, no empty spaces

Tablet:
→ No squished UI

Mobile:
→ No overflow
→ Buttons clickable
→ Text readable

-------------------------------------

STEP 15 — PERFORMANCE

Avoid:

- Heavy re-renders
- Large images

Ensure:
- Smooth scrolling
- Fast interaction

-------------------------------------

SUCCESS CRITERIA:

- Clean layout on all devices
- No overlapping UI
- No hidden dropdowns
- Fully accessible for visually impaired users
