Upgrade Dora Dungeons payment system to a production-grade Stripe integration using webhooks.

## GOAL

* Secure Stripe payment flow

* DO NOT trust frontend for payment success
* ONLY use Stripe webhooks to confirm payment
* DO NOT break existing game logic

## STEP 1 — INSTALL STRIPE SDK

Install:

npm install stripe

---

Add environment variables:

STRIPE_SECRET_KEY=your_secret_key
STRIPE_WEBHOOK_SECRET=your_webhook_secret

---

DO NOT expose secret key to frontend

---

## STEP 2 — CREATE CHECKOUT SESSION (SECURE)

Update:

POST /api/create-checkout-session

---

Use Stripe SDK:

* mode: "payment"
* line_items:

  * name: "Dora Dungeons — Full Access"
  * amount: $30
  * quantity: 1

---

IMPORTANT:

Attach metadata:

metadata: {
userId: currentUser.id
}

---

Set URLs:

success_url: https://your-app.com/payment-success
cancel_url: https://your-app.com/payment-cancel

---

Return:

{ url: session.url }

---

## STEP 3 — CREATE STRIPE WEBHOOK ENDPOINT

Create:

POST /api/stripe-webhook

---

IMPORTANT:

* Use raw body (DO NOT JSON parse before verification)
* Verify signature using STRIPE_WEBHOOK_SECRET

---

Handle event:

checkout.session.completed

---

Inside handler:

1. Extract:

session.metadata.userId

2. Find user in DB

3. Set:

player.hasPaid = true

4. Save to DB

---

IMPORTANT:

* Make operation idempotent
* If already paid → do nothing

---

## STEP 4 — REMOVE INSECURE PAYMENT SUCCESS LOGIC

REMOVE:

/api/payment-success endpoint (or disable logic)

---

REPLACE WITH:

Frontend-only success page (no DB mutation)

---

This page should:

* Say "Payment successful"
* Redirect back to game

On game load (GET /game/state):

* player.hasPaid comes from DB

---

Frontend:

IF:

player.hasPaid === true

→ allow next level

---

IF false:

→ trigger payment flow again
