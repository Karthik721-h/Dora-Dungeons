You are a senior backend engineer specializing in game state persistence and recovery systems.

Implement a robust SAVE / RESTORE system for Dora Dungeons.

---

## GOAL

* Persist full game state safely
* Restore state perfectly on reload
* Prevent data loss
* Maintain voice-first continuity

---

## STRICT RULES

* DO NOT break existing game logic
* DO NOT duplicate state between frontend and backend
* Backend (DB) is the ONLY source of truth
* Save AFTER every meaningful action

---

Ensure the following player fields are ALWAYS persisted:

* id
* email
* dungeonLevel
* dungeonLevelCompleted
* hasPaid

---

Combat + stats:

* hp / maxHp
* mp / maxMp
* attack / defense
* statusEffects

---

Progress:

* currentRoomId
* exploredRooms
* enemies (with HP state)
* dungeon seed

---

Inventory:

* gold
* weapons (IDs)
* armors (IDs + levels)
* inventory items

---

Game state:

* gameStatus (EXPLORING, IN_COMBAT, VICTORY, GAME_OVER)
* logs (last 80 lines max)

Ensure saveSession() is called AFTER:

* every command (/game/command)
* shop actions (buy/sell/upgrade)
* level transitions (/next-level, /replay-level)
* restart after death
* payment completion (webhook)

---

NO action should mutate state without saving
